# TypeScript Vite Vue Tailwind DaisyUI Development Specs

Guidelines for TypeScript development with Vite, Vue.js, Tailwind CSS, and DaisyUI.

## Overview

This specification provides comprehensive coding guidelines and best practices for front-end development using TypeScript, Vite, Vue.js, Tailwind CSS, and DaisyUI. It covers code style, performance optimization, Vue component patterns, and build optimization strategies.

## Included Rules

- **[Code Style and Structure](code-style-and-structure.md)** - Consistent code style and structure across the project
- **[General TypeScript Guidelines](general-typescript.md)** - TypeScript best practices including interfaces and functional components
- **[Performance Optimization](performance-optimization.md)** - Lazy loading, image optimization, and Web Vitals
- **[Vite Build Optimization](vite-build-optimization.md)** - Build process optimization strategies including chunking and code splitting
- **[Vue.js Component Guidelines](vue-js-component.md)** - Component style, naming conventions, and Composition API usage
- **[VueUse Library](vueuse-library.md)** - Leveraging VueUse functions for reactivity and performance

## Usage

These `.md` files provide comprehensive coding guidelines and best practices for TypeScript, Vite, Vue.js, Tailwind CSS, and DaisyUI development. They can be used as reference documentation for AI-assisted development or integrated into project documentation.

## Key Principles

- Write concise, maintainable TypeScript code
- Use functional and declarative programming patterns
- Prefer Composition API with script setup style
- Implement responsive design with mobile-first approach
- Optimize for performance and Web Vitals
- Leverage VueUse for enhanced reactivity
